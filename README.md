# xxbot

## 项目概述

桌面交互式机器人，具有以下功能：
- 捕获硬件/软件层面的各种信息
- 主动交互（移动、说话等）
- 自主学习、记录，形成特定的性格

## 系统需求

### 硬件需求
- 简单的移动舵机驱动
- 音频收集与分析模块
- 扬声器
- 液晶电容屏驱动
- WIFI功能

### 软件需求
- 信息捕获触发机制
- 自主学习、记录，形成特定的性格（上位机运行即可）
- 液晶屏动画显示

## 硬件选型

- **主控芯片**：ESP S3

## 软件架构

### ESP S3 端

#### 应用层
- **RTOS框架**
  - LVGL_handler
  - 音频收集、分析
  - 综合处理
  - 中断系统
    - 接收MQTT信息，并作出处理（在综合处理中）

#### 驱动层
- 舵机驱动
- 音频接收
- 扬声器
- LVGL动画框架
- MQTT
  - 发送捕获信息给云服务器

### 云服务器
- MQTT信息转运

### 上位机
- **Python**
  - **应用层**
    - 接受MQTT信息，并进行综合处理和返回
  - **工具层**
    - MQTT收取
    - API请求
    - ESP - Skills
    - 长期记忆库
      - 长期记忆
      - 性格塑造

## 运作逻辑

1. **信息捕获**：ESP S3通过各种传感器和接口捕获硬件/软件层面的信息
2. **数据传输**：将捕获的信息通过MQTT协议发送到云服务器
3. **信息处理**：上位机通过MQTT接收信息，进行综合处理和分析
4. **决策生成**：基于处理结果和长期记忆库，生成相应的决策和响应
5. **指令下发**：将决策通过MQTT发送回ESP S3
6. **执行响应**：ESP S3执行相应的动作，如移动、说话、显示动画等
7. **学习迭代**：系统根据交互结果进行自主学习，不断完善性格模型

## 技术特点

- **模块化设计**：清晰的分层架构，便于维护和扩展
- **实时响应**：基于RTOS框架，保证系统的实时性
- **自主学习**：通过长期记忆库和性格塑造机制，实现机器人的个性化
- **远程控制**：通过MQTT和云服务器，实现远程监控和控制
- **多模态交互**：支持音频、视觉等多种交互方式
